# Cloudflare R2 Store Adapter (`@memofs/adapter-r2`)

The `@memofs/adapter-r2` adapter implements the core `MemoryStore` interface using Cloudflare R2 Object Storage. This is the foundation for distributed cloud-sync operations in Memo FS.

---

## Installation

```bash
npm install @memofs/adapter-r2
```

---

## Usage

Bind your R2 bucket to the Worker environment and initialize the store client:

```ts
import { Tekmemo } from "@memofs/core";
import { createR2BlobClient } from "@memofs/adapter-r2";

export default {
  async fetch(request, env) {
    const memo = new Tekmemo({
      projectId: "r2-project",
      // Bind to Cloudflare R2 Bucket
      store: createR2BlobClient({
        bucket: env.MY_R2_BUCKET,
      }),
    });
    
    // ... handling fetch ...
  }
}
```

---

## Configuration API (`R2BlobClientConfig`)

| Option | Type | Required | Description |
|---|---|---|---|
| `bucket` | `R2Bucket` | Yes | The Cloudflare R2 bucket binding object. |
| `prefix` | `string` | No | Optional key prefix to isolate multiple projects in one bucket. |
| `contentType`| `string` | No | Default content-type header for written blobs (default: `text/plain`). |
